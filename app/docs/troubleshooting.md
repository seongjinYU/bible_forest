# 운영 트러블슈팅 기록

> 실유저 대상 운영 중 발생한 이슈를 기록합니다. 발생 시점이 빠른 순으로 위에 쌓습니다.
> 포트폴리오/이력서용으로 재사용할 수 있도록, 증상 → 원인 → 해결 → 회고 순서로 작성합니다.

---

## 작성 가이드

각 항목은 아래 템플릿을 복사해서 채웁니다.

```md
### [YYYY-MM-DD] 한 줄 제목

- **영역**: FE / BE
- **심각도**: 🔴 치명(서비스 중단·데이터 손상) / 🟡 주요(일부 기능 오작동) / 🟢 경미(UX 불편)
- **증상**: 사용자/운영자가 무엇을 보고 이상하다고 느꼈는지
- **원인**: 근본 원인 (코드 위치 포함, 가능하면 `파일:줄`)
- **해결**: 어떻게 고쳤는지 (커밋/PR 링크 가능하면 첨부)
- **재발 방지**: 같은 문제를 막기 위해 추가한 장치(테스트, 모니터링, 코드 리뷰 체크포인트 등)
- **회고**: 더 빨리 알아챌 수 있었던 방법, 다음에 비슷한 설계를 할 때 주의할 점
```
